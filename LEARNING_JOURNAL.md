# Learning Journal

## Session 001 - 项目立项

日期：2026-03-12

### 我们这一步在做什么

我们没有急着做一个“很强的 Agent”，而是先把目标收敛到一个最小可用版本：

- 做一个带桌面宠物外壳的个人助理
- `v1` 先专注于本地文件检索

### 这一步学到的东西

- `Agent` 不等于“一个会聊天的模型”
- 一个可用的 Agent 至少要有：
  - 明确目标
  - 可调用的工具
  - 可追溯的数据来源
  - 清楚的边界
- 桌面宠物只是表现层，不是核心能力本身
- 真正有价值的能力是：理解意图、找到信息、把结果可靠地交给用户

### 我们做出的关键决定

- `v1` 先不碰密码保存
- `v1` 不扫描整个电脑
- `v1` 只处理用户指定目录
- 技术路线先定为：
  - Electron 做桌面端
  - Python 做 Agent 服务
  - SQLite 做本地存储和检索基础

### 为什么这些决定重要

- 先收窄范围，项目更容易真正做完
- 先解决“找得到文件”这个刚需，比先做花哨交互更有价值
- 安全边界一开始就立住，后面不容易走偏

## Session 002 - 固定工作目录与远程仓库

日期：2026-03-12

### 我们这一步在做什么

把学习项目的本地目录和远程 Git 仓库固定下来，避免后面代码、笔记、版本历史分散在不同地方。

### 这一步学到的东西

- 做长期项目时，尽早固定工作目录很重要
- “共同笔记”和“代码仓库”最好在同一个项目目录里
- 本地 Git 仓库和远程 GitHub 仓库是两件事：
  - 本地仓库负责当前改动管理
  - 远程仓库负责同步、备份和协作
