# Python 索引服务教学流程

这份文档记录我准备怎样带你学当前这版 `Python` 索引服务，不求一口气学完，而是按“先能看懂，再能改，再能自己扩展”的顺序推进。

## 教学目标

- 让你看懂当前 `v1` 骨架为什么这样拆
- 让你能自己跑通“扫描目录 -> 建索引 -> 搜索”
- 让你能自己完成一两个小扩展
- 为后面接 `Electron` 和更复杂检索打基础

## 教学顺序

### 第 1 步：先看全局，不急着钻细节

目标：

- 知道项目现在有哪些模块
- 知道每个模块分别负责什么

我会带你看这些文件：

- `src/index_service/api.py`
- `src/index_service/indexing.py`
- `src/index_service/storage.py`
- `src/index_service/search.py`

这一步你要建立的核心认知是：

- `api.py` 管 HTTP 入口
- `indexing.py` 管扫描、读取、切块、建索引
- `storage.py` 管 SQLite 和 FTS5
- `search.py` 管查询服务封装

### 第 2 步：专门讲存储层

目标：

- 理解为什么要分普通表和 FTS 表
- 理解 FTS5 在这里到底扮演什么角色

会重点讲：

- `files` 表存什么
- `document_chunks` 表存什么
- 为什么文本要切块再建索引
- FTS5 触发器为什么存在

学完后你应该能回答：

- 为什么不是把整篇文件直接扔进一个字段就完事
- 为什么搜索结果里要保留路径、块编号、行号

### 第 3 步：再讲索引流程

目标：

- 理解一个目录是怎么一步步进入数据库的

会重点讲：

- 根目录校验
- 文件过滤策略
- UTF-8 文本读取
- 切块逻辑
- 整文件替换更新的取舍

学完后你应该能自己改：

- 允许的文件扩展名
- 跳过的目录名
- 分块大小和重叠长度

### 第 4 步：最后讲 API

目标：

- 看懂 FastAPI 在这个项目里只是“薄入口”

会重点讲：

- `GET /healthz`
- `POST /api/v1/index`
- `POST /api/v1/search`
- 请求模型和响应模型
- 400 错误是怎么抛出来的

学完后你应该能自己加：

- 一个 `GET /api/v1/roots`
- 一个 `DELETE /api/v1/index`

### 第 5 步：边讲边做小练习

目标：

- 不只听懂，还要能自己改出结果

我会优先给你这种练习：

- 新增一种允许索引的扩展名
- 改搜索结果返回字段
- 给索引结果增加统计信息
- 给查询加一个简单文件名匹配

这些练习刻意控制得很小，是为了让你每次只跨一个台阶。

### 第 6 步：开始做第一轮扩展

等你能稳定看懂当前骨架后，再进入下一层：

- 增量索引
- 忽略规则配置化
- 多编码兼容
- 文档摘要
- Electron 调用链路

## 我带你的方式

- 不会一开始塞太多概念
- 先讲当前代码里已经存在的东西
- 每次讲完都配一个小改动
- 你改不动时，我会先指出应该看哪一层，而不是直接把答案糊给你

## 当前最近的教学起点

如果下一次开始讲，我会从 `storage.py` 开始，因为这是这版服务最容易“看上去简单，实际上决定后续上限”的部分。
